git常用命令

发表于 2017-06-21 |

git学习总结

本文总结了git常用的命令及其作用

版本控制

  1. git status 命令查看仓库当前的状态
  2. git diff file 查看某个文件的修改
  3. git reset –hard commit_id 到某个版本
    commit_id 一般写前五位即可
  4. git log 可以查看提交历史
    只能看这个版本之前的版本
  5. git reflog查看命令历史，能看到全部版本，以便确定要回到未来的哪个版本
    比如 commit reset命令，和每个命令对应的版本号以及备注
  6. git checkout – file 把文件在工作区的修改全部撤销
    未添加到暂存区，直接丢弃工作区的修改，回到上次add或者上次上次commit
  7. git reset HEAD file 可以把暂存区的修改撤销掉（unstage），重新放回工作区
    已经添加到暂存区，把add进缓存区的修改撤销掉，可以配合上个命令继续把工作区的修改也丢弃。
  8. git rm file 删除文件
    若确定要删除，先用这个命令，然后commit
    若删除错误，由于版本库还有这个文件，可以用git checkout – file 恢复到最新版本

一些提示信息：
Changes not staged for commit 暂存区没东西
Changes to be committed 暂存区有东西，未提交到版本库

远程仓库

  1. git remote add origin git@github.com:yungegege/learngit.git 关联远程库
  2. git push -u origin master 推送
    第一次推送master分支时，加上了-u参数，Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令
    之后就可以用git push origin master推送了
  3. git clone git@github.com:yungegege/gitskills.git 克隆远程库

分支管理

  1. git checkout -b dev 创建并切换分支
    -b是创建并切换
    相当于git branch dev 创建
    git checkout dev 切换
  2. git merge dev 合并分支
  3. git branch -d dev 删除分支
    如果分支还没有被合并，如果删除，将丢失掉修改，如果要强行删除，需要使用大写的-D参数
  4. git branch 查看有哪些分支
  5. git log –graph –pretty=oneline –abbrev-commit 查看分支合并情况
  6. git merge dev 合并dev分支
    默认是Fast forward模式，但这种模式下，删除分支后，会丢掉分支信息。
  7. git merge –no-ff -m “merge with no-ff” dev 合并dev分支，–no-ff参数表示禁用Fast forward
    强制禁用Fast forward模式，Git就会在merge时生成一个新的commit，这样，从分支历史上就可以看出分支信息。
    合并分支时，加上–no-ff参数就可以用普通模式合并，合并后的历史有分支，能看出来曾经做过合并，而fast forward合并就看不出来曾经做过合并
  8. git stash 可以把当前工作现场“储藏”起来，等以后恢复现场后继续工作
  9. git stash list 查看保存的工作现场
  10. git stash pop 恢复指定的工作现场，并删除这个现场内容。
    如果现场就一个，后面不用跟其他单词，如果有的多个就要跟上现场的名字。
    相当于git stash apply恢复，但是恢复后，stash内容并不删除;在加上git stash drop来删除stash内容。
  • add并commit的是正常的普通的dev分支，切换分支会隐藏
  • 没有add或者有add但未commit的是stash的使用条件，切换分支也会隐藏

远程分支

  1. git checkout -b dev origin/dev 在本地创建和远程分支对应的分支
  2. git push origin dev 推送分支
  3. git branch –set-upstream-to=origin/dev dev 设置本地分支与远程分值的连接
  4. git pull 把关联的远程分支拉下来
  5. git remote -v 查看远程库信息
