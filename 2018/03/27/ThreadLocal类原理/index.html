ThreadLocal类原理

概念

为什么要引入ThreadLocal类?
为了解决线程共享变量导致不安全的问题，引入了ThreadLocal类。对于一个变量，他给每个线程创建不同的副本，各个线程互不影响，用的是空间换时间。
ThreadLocal类维护了一个ThreadLocalMap，ThreadLocalMap底层和HashMap类似，有一个Entry数组，key是ThreadLocal，value是当前线程要取的值。

get方法

  1. 可以看到，每个线程都有一个ThreadLocalMap;
  2. ThreadlLocalMap里面存了一个Entry数组，可以根据当前的ThreadLocal找到数组中对应的Entry的位置；
  3. Entry的key是ThreadLocal，因为这个Entry已经是当前ThreadLocal对应的Entry了，直接通过Entry.value获取到要get的值。
    一个线程对应一个ThreadLocalMap，一个ThreadLocalMap里面存了多个Entry(ThreadLocal<?> k, Object v组成的数组。也就是一个线程有多个ThreadLocal来存储数据。

set方法

  1. 获取当前线程的ThreadLocalMap;
  2. 若ThreadLocalMap不为空，调用ThreadLocalMap的set方法;
  3. 若ThreadLocalMap为空，调用ThreadLocal的createMap方法创建一个ThreadLocalMap;

remove方法

  1. 获取当前线程的ThreadLocalMap;
  2. 如果ThreadLocalMap不为空，调用ThreadLocalMap的remove方法;

应用举例

各个线程修改的都是都是本线程所对应的变量的副本，并不影响别的线程，代码一开始为什么要重写initialValue方法呢？
当用get获取值时，如果当前线程对应的ThreadLocalMap为空，那么会调用setInitialValue()方法，这个方法里面调用会initialValue()方法初始化值，如果不重写，直接get就get不到，然后在创建一个ThreadLocalMap对象把初始化的值放进去。
