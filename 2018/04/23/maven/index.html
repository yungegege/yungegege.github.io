maven

发表于 2018-04-23 |

maven是基于项目对象模型(POM),可以通过一小段描述信息来管理项目的构建、报告和文档的软件项目管理工具。

安装maven

点击这里进入maven下载页面
点图中箭头所示位置进行下载

下载解压后的目录

bin目录包含mvn的运行脚本
boot目录包含一个类加载器的框架，maven用它加载自己的类库
conf模板包含配置文件
lib目录包含maven运行时用到的所有类库，包括自身的和第三方的

配置环境变量

M2_HOME：E:\maven\apache-maven-3.5.0
PATH：E:\maven\apache-maven-3.5.0\bin;

打开cmd输入mvn -v能查到maven和java版本号就安装成功了

命令

mvn -v 查看版本
mvn compile 编译
mvn test 单元测试
mvn package 把本目录文件打包成jar包
mvn clean 删除target文件夹(编译生成的字节码文件和测试报告)
mvn install 把打包好的jar包安装到本地仓库中，然后别的项目在pom.xml中引入后就可以导入这个jar包的类了

maven自动建立目录骨架

第一种

在cmd中输入命令：mvn archetype:generate 回车,
让选择maven-archetype-quickstart version: 选6或者7
然后输入groupId、artifactId、version、package
就能成功创建目录

第二种

mvn archetype:generate -DgroupId=公司网址的反写.项目名 -DartifactId=项目名-模块名 -Dversion=版本号 -Dpackage=代码所在包名

maven中的坐标和仓库

坐标 构件
仓库 本地仓库和远程仓 库
镜像仓库 修改setting.xml文件
更改仓库位置 修改setting.xml文件
setting.xml文件位于maven安装目录下的conf文件夹里面

在idea中创建maven项目

  1. file -> new -> project
  2. 在弹出的new project 页面，选择maven -> 勾选Create from artifactype,选择maven-archetype-webapp,next
  3. 填写groupId 和archetypeId，next
  4. 选择Maven home directory，next
  5. 这个界面，如果这个项目只有这一个module不用改，直接finish

maven+ssm框架

下图是一个maven项目的目录，src下面还可以有个test文件夹，里面放测试代码，本项目目前没有测试

主要配置以下几个文件

  1. applicationContext-dao.xml 配置数据源、SqlSessionFactory、mapper扫描器。
  2. applicationContext-service.xml 配置service接口。
  3. applicationContext-transaction.xml 事务管理。
  4. sprintmvc.xml springmvc的配置，配置处理器映射器、适配器、视图解析器(这里我们统一采用注解的方式进行开发)。
  5. SqlMapConfig.xml mybatis的配置文件，配置别名、settings、mapper。

注解的用法

applicationContext-service.xml文件里如果没有<context:component-scan base-package="com.demo.ssm.service"/>这句，就要通过标签方式注入，例如：<bean id="xxxService" class="service.impl.xxxServiceImpl"/>
通过@Autowired注解完成service的依赖注入，通过@Controller注解将Controller自动添加到spring容器IOC中，通过@RequestMapping(“/queryItems”)注解指明访问该Controller的url。
